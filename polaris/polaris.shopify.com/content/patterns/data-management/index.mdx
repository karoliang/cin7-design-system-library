## Core Concepts

### Domain-Driven Design

Structure your application around business domains:

```typescript
// Domain entities
interface Product extends BaseEntity {
  name: string;
  price: number;
  stock: number;
}

// Repository for data access
class ProductRepository extends BaseRepository<Product> {
  // Implementation
}

// Use case for business logic
class ProductUseCase extends BaseUseCase {
  constructor(private repository: ProductRepository) {
    super();
  }
  
  async updateStock(id: string, quantity: number): Promise<void> {
    // Business logic here
  }
}
```

### State Management

Use Zustand for cross-layer state management:

```typescript
import { create } from 'zustand';

export const useAppStore = create((set, get) => ({
  products: [],
  loading: false,
  
  loadProducts: async () => {
    set({ loading: true });
    const products = await productRepository.findAll();
    set({ products: products.data, loading: false });
  }
}));
```

### Data Flow

1. **UI Layer** triggers actions
2. **Use Cases** handle business logic
3. **Repositories** manage data access
4. **Store** maintains application state
5. **Events** notify other layers
