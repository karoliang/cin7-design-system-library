# {frontmatter.title}

<Lede>

Bar charts are ideal for comparing values across categories, showing rankings, and visualizing discrete data. Use vertical columns for time-based categories or horizontal bars for long category names.

</Lede>

## React Component

```tsx
import { BarChart } from '@cin7/highcharts-adapter/react';

function ProductSales() {
  return (
    <BarChart
      title="Sales by Product Category"
      orientation="vertical"
      series={[
        {
          name: 'Q1 2025',
          data: [450, 320, 580, 410, 290]
        },
        {
          name: 'Q2 2025',
          data: [510, 380, 620, 450, 340]
        }
      ]}
      xAxis={{
        categories: ['Electronics', 'Clothing', 'Food', 'Books', 'Toys']
      }}
      yAxis={{
        title: { text: 'Sales ($K)' }
      }}
    />
  );
}
```

## Props

### Required props

| Prop | Type | Description |
|------|------|-------------|
| `series` | `SeriesBarOptions[] \| SeriesColumnOptions[]` | Array of data series to display |

### Optional props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Chart title |
| `subtitle` | `string` | - | Chart subtitle |
| `xAxis` | `XAxisOptions` | `{}` | X-axis configuration |
| `yAxis` | `YAxisOptions` | `{}` | Y-axis configuration |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Bar direction |
| `dataLabels` | `boolean` | `false` | Show data labels on bars |
| `stacking` | `'normal' \| 'percent'` | - | Enable data stacking |
| `grouping` | `boolean` | `true` | Enable grouping for multiple series |
| `legend` | `boolean` | `true` | Show legend |
| `tooltip` | `boolean \| TooltipOptions` | `true` | Configure tooltips |
| `theme` | `Cin7ChartTheme` | `{ mode: 'light' }` | Theme configuration |
| `height` | `number \| string` | `400` | Chart height |
| `width` | `number \| string` | `'100%'` | Chart width |

## Variations

### Vertical columns

Standard vertical bar chart for time-based or ordered categories:

```tsx
<BarChart
  orientation="vertical"
  series={[{ name: 'Revenue', data: [100, 150, 120, 180] }]}
  xAxis={{ categories: ['Q1', 'Q2', 'Q3', 'Q4'] }}
/>
```

### Horizontal bars

Better for long category names or ranking visualization:

```tsx
<BarChart
  orientation="horizontal"
  series={[{ name: 'Market Share', data: [45, 30, 15, 10] }]}
  xAxis={{
    categories: [
      'Very Long Product Name A',
      'Very Long Product Name B',
      'Very Long Product Name C',
      'Very Long Product Name D'
    ]
  }}
/>
```

### Stacked bars

Show composition and total simultaneously:

```tsx
<BarChart
  stacking="normal"
  series={[
    { name: 'Product A', data: [50, 60, 70, 80] },
    { name: 'Product B', data: [30, 40, 50, 60] },
    { name: 'Product C', data: [20, 30, 40, 50] }
  ]}
/>
```

### Percentage stacked

Compare proportions across categories:

```tsx
<BarChart
  stacking="percent"
  series={[
    { name: 'Desktop', data: [60, 55, 50, 45] },
    { name: 'Mobile', data: [30, 35, 40, 45] },
    { name: 'Tablet', data: [10, 10, 10, 10] }
  ]}
/>
```

### Grouped bars

Compare multiple series side-by-side:

```tsx
<BarChart
  grouping={true}
  series={[
    { name: '2024', data: [100, 120, 140, 160] },
    { name: '2025', data: [120, 150, 180, 200] }
  ]}
  xAxis={{ categories: ['Q1', 'Q2', 'Q3', 'Q4'] }}
/>
```

### With data labels

Display exact values on bars:

```tsx
<BarChart
  dataLabels={true}
  series={[{ name: 'Sales', data: [45000, 52000, 61000, 58000] }]}
/>
```

## Best practices

Bar charts should:

- **Compare discrete categories**: Ideal for non-continuous data
- **Order meaningfully**: Sort by value, time, or logical sequence
- **Use consistent bar widths**: Maintain visual balance
- **Provide context**: Include units in axis labels (e.g., "Revenue ($K)")
- **Choose orientation wisely**: Vertical for time, horizontal for long names

Bar charts should not:

- Show trends over time with many data points (use line charts instead)
- Display part-to-whole relationships (use pie charts instead)
- Have too many categories (causes visual clutter)
- Use 3D effects (distorts values and reduces clarity)

## Accessibility

Bar charts include:

- ARIA labels describing the comparison purpose
- Keyboard navigation for interactive elements
- High contrast colors for better visibility
- Screen reader-compatible data tables

Provide meaningful `ariaLabel` prop:

```tsx
<BarChart
  ariaLabel="Sales comparison showing Product C leading at $580K followed by Product A at $450K"
  // ... other props
/>
```
