# Color Tokens

<Lede>Color tokens provide semantic meaning to color values, ensuring consistency and accessibility across your application.</Lede>

<TokensNav selected="color" />

## Understanding Color Tokens

Color tokens in Cin7 DSL are organized by their semantic purpose rather than their visual appearance. This approach ensures:

- **Consistency**: Same semantic meaning across all components
- **Accessibility**: Automatic contrast ratios for text and backgrounds
- **Themability**: Easy switching between light/dark themes
- **Maintainability**: Change colors globally by updating tokens

## Implementation Examples

### CSS Usage

```css
.product-card {
  background: var(--p-color-bg-surface);
  border: 1px solid var(--p-color-border);
  color: var(--p-color-text);
}

.product-card:hover {
  background: var(--p-color-bg-surface-hover);
  border-color: var(--p-color-border-hover);
}

.product-price {
  color: var(--p-color-text-success);
}

.out-of-stock {
  color: var(--p-color-text-critical);
}
```

### React Component Usage

```jsx
import { Box, Text } from '@shopify/polaris';

function StatusIndicator({ status }) {
  const toneMap = {
    active: 'success',
    pending: 'warning',
    error: 'critical',
    info: 'info'
  };
  
  return (
    <Box 
      background={`bg-fill-${toneMap[status]}`}
      padding="200"
      borderRadius="100"
    >
      <Text tone={toneMap[status]}>{status}</Text>
    </Box>
  );
}
```

### ExtJS Integration

```javascript
Ext.define('MyApp.view.StatusPanel', {
  extend: 'Ext.panel.Panel',
  
  setStatus: function(status) {
    const colorMap = {
      success: 'var(--p-color-bg-fill-success)',
      warning: 'var(--p-color-bg-fill-warning)',
      error: 'var(--p-color-bg-fill-critical)'
    };
    
    this.setStyle({
      backgroundColor: colorMap[status] || 'var(--p-color-bg-surface)'
    });
  }
});
```

## Best Practices

1. **Use semantic tokens**: Choose tokens based on purpose, not appearance
2. **Avoid hardcoding**: Never use hex values directly in components
3. **Test in all themes**: Ensure your UI works in light and dark modes
4. **Consider context**: Use appropriate color intensity for the importance
5. **Maintain consistency**: Use the same token for similar purposes
