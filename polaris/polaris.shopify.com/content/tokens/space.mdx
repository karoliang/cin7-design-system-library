# Space Tokens

<Lede>Space tokens provide a consistent spacing system that creates visual rhythm and hierarchy throughout your application.</Lede>

<TokensNav selected="space" />

## Understanding Space Tokens

The Cin7 DSL spacing system is based on a 4px base unit, creating a harmonious scale that works across all screen sizes and components. This systematic approach ensures:

- **Visual Consistency**: Predictable spacing throughout the UI
- **Responsive Design**: Scales appropriately on different devices
- **Easy Maintenance**: Change spacing globally by updating tokens
- **Better Collaboration**: Designers and developers use the same values

## Implementation Examples

### CSS Usage

```css
/* Component spacing */
.card {
  padding: var(--p-space-400);
  margin-bottom: var(--p-space-300);
}

/* Inline elements */
.button-group {
  display: flex;
  gap: var(--p-space-200);
}

/* Section spacing */
.page-section {
  padding-top: var(--p-space-800);
  padding-bottom: var(--p-space-800);
}

/* Responsive spacing */
@media (max-width: 768px) {
  .page-section {
    padding-top: var(--p-space-600);
    padding-bottom: var(--p-space-600);
  }
}
```

### React Component Usage

```jsx
import { BlockStack, InlineStack, Box } from '@shopify/polaris';

function ProductCard({ product }) {
  return (
    <Box padding="400" background="surface" borderRadius="200">
      <BlockStack gap="300">
        <img src={product.image} alt={product.name} />
        
        <BlockStack gap="100">
          <Text variant="headingMd">{product.name}</Text>
          <Text variant="bodySm" tone="subdued">{product.category}</Text>
        </BlockStack>
        
        <InlineStack gap="200" align="space-between">
          <Text variant="headingLg">${product.price}</Text>
          <Button size="slim">Add to Cart</Button>
        </InlineStack>
      </BlockStack>
    </Box>
  );
}
```

## Best Practices

1. **Use the spacing scale**: Stick to defined tokens rather than arbitrary values
2. **Be consistent**: Use the same spacing for similar UI patterns
3. **Consider context**: Tighter spacing for related items, looser for separate sections
4. **Test responsively**: Ensure spacing works on all screen sizes
5. **Avoid mixing units**: Use tokens instead of mixing px, rem, or em values
