# Enhanced Include System

<Lede>
Advanced features that take the revolutionary include system to the next level with design tokens, type-safe builders, theme support, and enterprise-grade capabilities.
</Lede>

The enhanced include system builds upon the revolutionary foundation to provide enterprise-grade features for modern application development.

## Type-Safe Builder Pattern

Get full IntelliSense support and compile-time type checking with the enhanced builder pattern.

### Basic Usage

```typescript
import { createTypedIncludeBuilder, DesignTokens, Presets } from '@cin7/include-system';

// Create a builder with default configuration
const builder = createTypedIncludeBuilder({
  designTokens: {
    theme: 'light',
    spacing: 'base'
  }
});

// Type-safe includes with full autocomplete
const includes = [
  builder.react.Card.default({
    designTokens: DesignTokens.brand,
    sectioned: true
  }),
  builder.react.Button.primary({
    designTokens: Presets.primaryButton,
    onClick: 'handleClick'
  }),
  builder.extjs.DataGrid.enterprise({
    features: ['grouping', 'filtering', 'export']
  })
];
```

### Preset Configurations

```typescript
// Use built-in presets for common patterns
const quickIncludes = [
  // Primary button with brand theming
  builder.react.Button.primary(Presets.primaryButton),

  // Success badge with optimal sizing
  builder.react.Badge.success(Presets.successBadge),

  // Large data grid for dashboard use
  builder.extjs.DataGrid.enterprise(Presets.largeGrid)
];
```

## Design Token System

### Core Design Tokens

The enhanced include system includes a comprehensive design token system:

```typescript
include "react" "Card" "default" {
  designTokens: {
    theme: "dark",           // light, dark, brand, neutral
    size: "medium",          // small, medium, large, extra-large
    spacing: "loose",        // none, tight, base, loose, extra-loose
    elevation: "medium",     // none, small, medium, large, extra-large
    borderRadius: "medium",  // none, small, medium, large, extra-large
    colorScheme: "brand"     // Any valid color scheme
  }
}
```

### Design Token Presets

```typescript
import { DesignTokens } from '@cin7/include-system';

// Available presets
const themeTokens = {
  light: DesignTokens.light,
  dark: DesignTokens.dark,
  brand: DesignTokens.brand,
  neutral: DesignTokens.neutral
};

const sizeTokens = {
  small: DesignTokens.small,
  medium: DesignTokens.medium,
  large: DesignTokens.large
};

const spacingTokens = {
  tight: DesignTokens.tight,
  base: DesignTokens.base,
  loose: DesignTokens.loose
};
```

## Performance Considerations

### Design Token Caching

The enhanced include system automatically caches design token calculations:

```typescript
// Tokens are cached for optimal performance
const cachedIncludes = [
  include "react" "Card" "default" {
    designTokens: { theme: "dark", spacing: "base" }  // Cached
  },
  include "react" "Button" "primary" {
    designTokens: { theme: "dark", spacing: "base" }  // Reuses cached tokens
  }
];
```

### Lazy Loading Support

Configure components to load only when needed:

```typescript
include "extjs" "DataGrid" "enterprise" {
  designTokens: { theme: "dark" },
  lazy: true,  // Load on demand
  placeholder: "Loading data..."
}
```

## Migration from Basic Include System

### From Basic Syntax

```typescript
// Basic include (still supported)
include "react" "Card" "default"
```

### To Enhanced Syntax

```typescript
// Enhanced include with configuration
include "react" "Card" "default" {
  designTokens: {
    theme: "light",
    spacing: "base"
  },
  accessibility: {
    ariaLabel: "Card content"
  }
}
```

### Type-Safe Migration

```typescript
// Step 1: Import builder
import { createTypedIncludeBuilder } from '@cin7/include-system';

// Step 2: Create builder
const builder = createTypedIncludeBuilder();

// Step 3: Replace includes
// Old: include "react" "Card" "default"
// New: builder.react.Card.default()
```

## Best Practices

### 1. Use Design Tokens Consistently

```typescript
// Good: Use design tokens for theming
include "react" "Card" "default" {
  designTokens: Presets.secondaryCard
}

// Avoid: Hardcoded values
include "react" "Card" "default" {
  backgroundColor: "#ffffff"  // Avoid direct styling
}
```

### 2. Leverage Responsive Design

```typescript
// Good: Responsive configuration
include "react" "Button" "primary" {
  responsive: {
    small: { size: "small" },
    large: { size: "large" }
  }
}
```

### 3. Include Accessibility Features

```typescript
// Good: Include accessibility
include "react" "Button" "primary" {
  accessibility: {
    ariaLabel: "Submit form",
    describedBy: "form-help"
  }
}
```

### 4. Use Type-Safe Builders When Possible

```typescript
// Good: Type-safe with IntelliSense
const builder = createTypedIncludeBuilder();
const includes = [builder.react.Button.primary()];

// Acceptable: String-based includes for dynamic scenarios
const dynamicInclude = {
  language: "react",
  component: "Button",
  variation: "primary"
};
```
