## Overview

The Cin7 DSL API reference provides comprehensive documentation for all components, hooks, utilities, and TypeScript types available in the framework. This reference is automatically generated from the TypeScript definitions to ensure accuracy and completeness.

## Using the API Reference

### Component Documentation

Each component includes:
- **Props Interface**: Complete TypeScript interface with descriptions
- **Usage Examples**: Real-world code examples
- **Default Values**: Default prop values
- **Required Props**: Clearly marked required properties
- **Event Handlers**: Callback signatures and usage

### Type Safety

All components are fully typed with TypeScript:

```typescript
import { Button, ButtonProps } from '@cin7/dsl-components';

const MyButton: React.FC<ButtonProps> = (props) => {
  return <Button {...props} />;
};
```

### IntelliSense Support

When using Cin7 DSL in VS Code or other TypeScript-aware editors, you'll get:
- Auto-completion for all props
- Inline documentation
- Type checking
- Quick fixes and refactoring support

## API Versioning

The Cin7 DSL API follows semantic versioning:

- **Major**: Breaking changes to existing APIs
- **Minor**: New features and backwards-compatible changes
- **Patch**: Bug fixes and documentation updates

Current version: **1.0.0**

## ExtJS Integration

The API reference includes documentation for ExtJS component adapters:

```typescript
import { ExtDataGrid, ExtFormPanel } from '@cin7/dsl-extjs-adapters';
```

These components provide advanced functionality while maintaining consistency with the Polaris design system.

## Contributing to API Docs

The API documentation is generated from:
- TypeScript interfaces and types
- JSDoc comments in the source code
- MDX files for additional examples

To contribute:
1. Add JSDoc comments to your components
2. Include `@example` tags for usage examples
3. Run `pnpm generate-api-docs` to update the reference
